# BOSS 投递助手

这是一个 Chrome 扩展，用于在 BOSS 直聘职位列表页按关键词和城市自动向匹配岗位发送固定招呼语。

## 安装

1. 在 Chrome 打开 `chrome://extensions/`
2. 打开右上角“开发者模式”
3. 点击“加载已解压的扩展程序”
4. 选择本仓库根目录（其中包含 `manifest.json`）
5. 刷新已经打开的 BOSS 直聘页面

## 使用

面板只需要填写四项：

- 关键词：使用英文逗号 `,` 分隔（也兼容中文逗号）。每个词会作为一个独立的 BOSS 搜索目标。默认 `AI,算法,实习生`
- 城市：使用英文逗号 `,` 分隔（也兼容中文逗号）。扩展会读取 BOSS 官方城市代码，并执行全部“关键词 × 城市”组合。进入目标城市的官方搜索范围后，区、县、县级市和开发区岗位不需要在卡片中重复显示城市名。默认 `上海`
- 招呼语：支持 `{salutation}`、`{hr}`、`{company}`、`{title}`、`{location}`、`{salary}` 占位符
- 发送上限：默认 `0`，表示处理全部匹配岗位；填写正整数时按该数量停止，不再有 30 条硬上限

不需要手动点击扫描。点击“开始全自动”后，扩展会直接进入 BOSS 职位搜索页 `/web/geek/jobs`，依次执行全部“关键词 × 城市”组合，然后逐个打开匹配岗位、核对岗位与聊天对象、发送招呼并返回列表继续。每个组合会持续翻页或向下加载；明确出现“没有更多”或连续 3 次没有发现新岗位后，才切换到下一个组合。

每个新岗位正常会出现两条消息：第一条是点击“立即沟通”后由 BOSS 按站内招呼语设置自动发送的消息；第二条是扩展在进入对应聊天后发送的岗位模板。扩展的默认第二条为：`{salutation}，我对贵公司的{title}岗位很感兴趣，经验和岗位要求比较匹配，保证出勤率和实习时间。希望贵公司可以给我一个机会！`

`{salutation}` 只使用页面明确提供的信息，不根据姓名猜测性别。例如页面显示“杨女士”时生成“您好，杨女士”，显示“王 HR”时生成“您好，王HR”；只有姓名但没有明确称谓时使用中性的“您好，姓名老师”，无法可靠识别姓名时使用“您好，招聘负责人”。

## 安全规则

- 本地记录已经发送过的岗位，再次遇到同一岗位时直接跳过
- 职位只从 BOSS 主搜索结果列表提取，不把页面上的相似职位或异地推荐作为目标岗位
- 页面显示“继续沟通、已沟通、已投递”等状态时直接跳过
- BOSS 弹出“已向BOSS发送消息”时自动点击“继续沟通”，进入对应聊天后填入当前岗位生成的招呼语并点击发送
- “发送”按钮为标准按钮、带按钮样式的 `div` 或文字位于 `span` 内都能识别
- 点击发送后必须检测到输入框清空或聊天区出现同样文案才记录成功；随后关闭聊天并重新进入当前职位搜索，继续下一岗位
- 每次发送前核对岗位 URL、公司、HR 或岗位上下文
- 职位详情链接不会再被误判为沟通按钮；只点击文字明确为“立即沟通/打招呼/聊一聊”的可用按钮
- “立即沟通”的文字位于 `span`、带按钮样式的 `div` 或标准按钮中都可以识别；失败时状态区会显示具体停在哪一步
- 无法确认当前聊天对象属于目标岗位时立即停止，不发送
- 同一岗位 10 分钟内刚发送过第二条招呼时，清空草稿并只跳过当前岗位，然后继续下一个
- 某个岗位临时打开失败时只在本轮跳过；重新启动后可以再次尝试
- 处理完详情页最后一条后会先返回列表，再继续翻页或加载更多结果
- 缺少关键词或城市时拒绝启动，避免无条件群发
- 遇到登录、验证码、风控或页面结构无法识别时停止或跳过，不绕过网站限制

扩展不会自动上传简历文件，也不会绕过 BOSS 的登录、验证码、风控或频率限制。

## 开发与测试

需要 Node.js 18 或更高版本：

```bash
npm run verify
```

该命令检查 JavaScript 语法和扩展清单，并运行防重复、岗位匹配、收件人核对、跨页搜索及称呼生成相关的逻辑测试。GitHub Actions 会在每次推送和拉取请求时执行同一套验证。

## 隐私

扩展不包含第三方统计或外部数据上传。详细说明见 [PRIVACY.md](./PRIVACY.md)。

本项目与 BOSS 直聘官方无关联。使用自动化功能前请确认符合平台规则，并自行控制发送范围和频率。
